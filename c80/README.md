# 矢量白板 - Vector Whiteboard

基于 Node.js + IPFS + React + Yjs 的全栈协作矢量白板应用。

## 🎯 功能特性

### 核心绘图功能
- ✏️ **自由画笔**: 支持自由绘制路径
- ⬜ **矩形**: 绘制矩形图形
- ⭕ **圆形**: 绘制圆形图形
- 📝 **文本**: 添加文本注释
- 🧹 **橡皮擦**: 擦除绘制内容
- 🎨 **颜色选择**: 自定义画笔颜色
- 📏 **粗细调节**: 调节画笔粗细

### 协作同步
- 🔄 **CRDT实时同步**: 使用 Yjs 实现无冲突实时协作
- 🌐 **WebRTC P2P**: 点对点数据传输
- 👥 **多用户协作**: 支持多人同时编辑

### 房间机制
- 🏠 **创建房间**: 生成唯一房间 CID
- 🔗 **加入房间**: 通过 CID 加入现有房间
- 📋 **复制分享**: 一键复制房间 CID

### 版本历史
- 📜 **历史记录**: 完整记录所有操作历史
- ⏪ **时间旅行**: 支持回溯到任意历史版本
- 🔙 **回到最新**: 一键返回最新版本

### 导出功能
- 💾 **SVG导出**: 将白板内容导出为 SVG 矢量图

### 数据持久化
- 🚀 **IPFS网络**: 数据存储在 IPFS 网络上
- 📊 **OrbitDB**: 分布式数据库存储操作记录

## 🛠️ 技术栈

### 前端
- **React 18**: 用户界面框架
- **Vite**: 构建工具
- **Yjs**: CRDT 数据结构实现实时协作
- **y-webrtc**: WebRTC 提供程序

### 后端
- **Node.js**: 运行环境
- **Express**: Web 框架
- **Helia**: IPFS 实现
- **OrbitDB**: 分布式数据库

## 📖 使用说明

### 1. 创建或加入房间
- 打开应用后，点击 "Create New Room" 创建新房间
- 或者输入房间 CID 点击 "Join Room" 加入现有房间

### 2. 绘图工具
- 左侧工具栏选择绘图工具（画笔、矩形、圆形、文本、橡皮擦）
- 选择颜色和线条粗细
- 在画布上点击或拖拽进行绘制

### 3. 协作
- 复制房间 CID 分享给其他用户
- 所有用户的操作会实时同步

### 4. 历史回溯
- 点击历史按钮打开历史面板
- 点击任意历史节点查看当时的白板状态
- 点击 "Back to Latest" 返回最新状态

### 5. 导出
- 点击导出按钮将当前白板导出为 SVG 文件

## 🔧 API 接口

### 创建房间
```
GET /api/room/create
```
返回新房间的 CID

### 获取房间数据
```
GET /api/room/:cid
```
获取指定房间的所有记录

### 同步数据
```
POST /api/room/:cid/sync
```
提交操作到房间

### 获取历史记录
```
GET /api/room/:cid/history
```
获取房间的完整操作历史

## 🌟 核心特性详解

### Yjs CRDT 同步
Yjs 提供了 Conflict-free Replicated Data Types (CRDT)，确保：
- 无冲突合并：所有用户的修改能正确合并
- 离线支持：用户离线后再上线仍能同步
- 高性能：增量更新，低延迟

### IPFS + OrbitDB 持久化
- 数据分布式存储在 IPFS 网络
- OrbitDB 提供文档型数据库接口
- 每个房间对应一个独立的数据库
- 支持离线数据缓存

### WebRTC 点对点通信
- 浏览器之间直接建立 P2P 连接
- 数据直接在用户间传输，不经过服务器
- 支持 NAT 穿透和信令服务器

## 📝 注意事项

1. **IPFS 节点启动**: 首次启动可能需要较长时间初始化 IPFS 节点
2. **网络连接**: WebRTC 需要良好的网络环境建立 P2P 连接
3. **浏览器兼容性**: 推荐使用 Chrome、Firefox、Edge 等现代浏览器
4. **数据持久化**: OrbitDB 数据存储在本地 `./data` 目录
