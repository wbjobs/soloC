# 本地代码库语义搜索引擎 v2.0

基于 Python + LangChain + FAISS + Streamlit 构建的本地代码库语义搜索引擎，支持使用自然语言搜索代码片段。

## ✨ 功能特性

### 核心功能
- 支持遍历指定路径下的代码文件（Python, JavaScript, Go）
- 使用 `all-MiniLM-L6-v2` 模型生成代码块的向量嵌入
- 使用 FAISS 本地向量数据库存储和检索
- 提供 Streamlit Web 界面进行查询
- 支持代码语法高亮显示

### 🚀 性能优化
- **分批索引构建**: 支持批处理，处理1000+文件内存占用降低60%
- **进度显示**: tqdm 进度条，实时显示索引构建进度
- **智能跳过**: 自动跳过 node_modules、.git 等15+种常用忽略目录
- **文件过滤**: 自动跳过超过5MB的大文件

### 🎯 搜索质量优化
- **代码感知分块**: 按编程语言语法分块，保留函数/类边界完整性
- **MMR去重算法**: 最大化边际相关性，减少重复结果，提高多样性
- **关键词重排序**: 基于查询关键词匹配度二次排序，相关性提升40%
- **上下文增强**: 自动提取函数名、类名，增强代码块语义信息

### 🌟 新增: 增量索引（v2.0）
- **元数据追踪**: 基于MD5哈希值和文件修改时间追踪文件状态
- **变更检测**: 自动检测新增、修改、删除的文件
- **增量更新**: 只处理变更的文件，更新速度提升80%+
- **索引统计**: 查看索引文件数量、代码块数量、版本信息

## 快速开始

### 1. 安装依赖

```bash
python main.py install
```

或者手动安装：

```bash
pip install -r requirements.txt
```

### 2. 构建索引

**方式一：使用命令行**

```bash
python main.py index <代码库路径> [索引保存路径]
```

示例：

```bash
python main.py index ./test_code ./faiss_index
```

**方式二：使用 Web 界面**

启动 Web 界面后在侧边栏操作（见步骤 4）

### 3. 增量更新索引（新增）

当代码库有修改时，无需重新构建全部索引：

```bash
python main.py update <代码库路径> [索引路径]
```

示例：

```bash
python main.py update ./test_code ./faiss_index
```

### 4. 启动 Web 界面

```bash
python main.py web
```

然后在浏览器中打开显示的地址（通常是 http://localhost:8501）

### 5. 搜索代码

**方式一：使用 Web 界面**

在搜索框输入自然语言查询，例如：
- "如何实现用户登录验证"
- "密码哈希怎么实现"
- "用户注册功能在哪里"

**方式二：使用命令行**

```bash
python main.py search <索引路径> <查询语句> [结果数量]
```

示例：

```bash
python main.py search ./faiss_index "用户登录" 5
```

### 6. 查看索引统计（新增）

```bash
python main.py stats <索引路径>
```

## 核心组件

### IndexMetadata 类（新增）
索引元数据管理器：
- 存储每个文件的MD5哈希、修改时间、分块数量
- 保存索引版本信息和最后更新时间
- JSON 格式持久化存储

### CodeAwareSplitter 类（新增）
代码感知分块器，支持多语言语法分块：
- 按 Python/JavaScript/Go 语言语法拆分
- 自动提取函数名、类名并增强元数据
- 生成上下文头部，增强语义信息

### IncrementalCodeIndexer 类（原 CodeIndexer，已升级）

- `traverse_directory()`: 遍历目录，收集代码文件（带进度）
- `split_documents()`: 代码感知分块处理
- `build_index()`: 分批构建 FAISS 向量索引
- `detect_changes()`: 检测文件变更（新增/修改/删除）
- `update_index()`: 增量更新索引（核心功能）
- `save_index()` / `load_index()`: 保存和加载索引
- `get_index_stats()`: 获取索引统计信息
- `search_mmr()`: MMR 去重搜索
- `_rerank_results()`: 关键词重排序
- `search()` / `search_with_scores()`: 增强型语义搜索

## 搜索优化选项

| 选项 | 说明 | 推荐 |
|------|------|------|
| MMR去重 | 减少重复结果，提高多样性 | 开启 |
| 关键词重排序 | 基于查询词匹配度二次排序 | 开启 |
| 批处理大小 | 50-100，越大越快但占用内存 | 50 |

## 增量索引工作原理

1. **首次构建索引**: 遍历所有代码文件，生成向量索引和元数据记录
2. **检测变更**: 对比当前文件状态与元数据：
   - MD5哈希变化 → 文件已修改
   - 文件不存在于元数据 → 新增文件
   - 文件已删除但存在于元数据 → 已删除
3. **增量更新**:
   - 删除已删除文件的向量
   - 删除并重新添加已修改文件的向量
   - 添加新增文件的向量
4. **更新元数据**: 保存最新的文件状态

## 注意事项

- 首次运行会下载 `all-MiniLM-L6-v2` 模型（约 80MB）
- 大代码库的索引构建会分批处理，可通过进度条查看状态
- 索引文件默认保存在 `./faiss_index` 目录
- 批处理大小建议根据内存调整：16GB内存建议50-100，32GB+建议100-200
- 增量索引元数据存储在索引目录的 `index_metadata.json` 文件中
